# 🛡️ Бот-антиспам для комментариев Telegram-канала

Этот бот следит за **комментариями под постами вашего канала** и:

- 🚫 **явный спам** (реклама 18+, «пишите в личку», простыни про заработок) —
  удаляет и **банит** автора автоматически;
- ⚠️ **что-то подозрительное** — удаляет сообщение и присылает **вам в личку**
  кнопки «🚫 Забанить / ✅ Не спам», чтобы решение осталось за вами;
- ✅ **обычных людей не трогает** — и чем больше человек пишет нормальных
  сообщений, тем меньше бот его проверяет.

> Написано максимально подробно, для полного новичка. Просто идите по шагам
> сверху вниз. Если застряли — напишите мне, на каком шаге.

<a name="как-это-устроено"></a>
## 🧠 Как это устроено (важно понять)

В Telegram «комментарии под постами канала» — это на самом деле **отдельная
группа**, которая называется **«группа обсуждений»** и привязана к каналу.
Когда кто-то комментирует пост — он на самом деле пишет в эту группу.

**Бот не может модерировать сам канал напрямую.** Он модерирует именно эту
группу обсуждений. Поэтому план такой:

```
Ваш канал  ──привязан──►  Группа обсуждений (сюда пишут комментарии)
                                  ▲
                                  │ бот-администратор читает сообщения,
                                  │ удаляет спам, банит спамеров
                              🤖 наш бот
```

Всё, что нужно от вас: включить комментарии (если ещё не включены) и добавить
бота в эту группу администратором. Дальше он работает сам.

<a name="шаг-1"></a>
## Шаг 1. Создаём бота у @BotFather

1. Откройте Telegram и в поиске найдите **@BotFather** (с синей галочкой).
2. Нажмите **Start** и отправьте команду:
   ```
   /newbot
   ```
3. BotFather спросит **имя** бота (любое, например «Мой Антиспам»).
4. Потом спросит **username** — он должен заканчиваться на `bot`
   (например `moy_antispam_bot`).
5. В ответ BotFather пришлёт **токен** — длинную строку вида:
   ```
   123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   ⚠️ **Это пароль от бота. Никому не показывайте и не выкладывайте в интернет.**
   Скопируйте его — понадобится дальше.

---

<a name="шаг-2"></a>
## Шаг 2. Подключаем комментарии к каналу

*(Если под постами уже есть комментарии — пропустите шаг.)*

1. Откройте свой **канал** → ⚙️ **Управление каналом** → **Обсуждение**
   (Discussion).
2. Нажмите **Создать группу** (или выберите существующую). Telegram сам создаст
   привязанную группу обсуждений.
3. Готово: теперь под постами появилась кнопка «Комментировать».

---

<a name="шаг-3"></a>
## Шаг 3. Добавляем бота в группу обсуждений (админом)

1. Откройте **группу обсуждений** (не канал!).
2. **Название группы → Участники → Добавить** → найдите вашего бота по
   username (`moy_antispam_bot`) и добавьте.
3. Теперь сделайте бота **администратором**: **Управление группой →
   Администраторы → Добавить администратора → выберите бота**.
4. Включите ему минимум два права:
   - ✅ **Удаление сообщений** (Delete messages)
   - ✅ **Блокировка пользователей** (Ban users)

   Остальные права можно выключить — они не нужны.

> Без этих двух прав бот не сможет удалять спам и банить — это самая частая
> причина «бот ничего не делает».

<a name="шаг-4"></a>
## Шаг 4. Проверяем локально на компьютере (по желанию)

Этот шаг нужен, только если хотите сначала протестировать у себя. Если хотите
сразу на хостинг — переходите к [Шагу 5](#шаг-5).

Нужен установленный **Python 3.10+** ([скачать](https://www.python.org/downloads/)).

```bash
# 1) Скачайте эту папку telegram-spam-bot к себе на компьютер и зайдите в неё
cd telegram-spam-bot

# 2) (рекомендуется) создайте отдельное окружение
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3) установите зависимости
pip install -r requirements.txt

# 4) создайте файл настроек из примера
cp .env.example .env            # Windows: copy .env.example .env
# откройте .env в блокноте и впишите свой BOT_TOKEN

# 5) запустите
python bot.py
```

Если увидели строку `Бот запущен. Жду сообщения…` — всё работает. Напишите
в группе обсуждений тестовое «спам-сообщение» (например *«пишите в личку, могу
переслать»*) с другого аккаунта — бот должен его удалить.

Остановить бота: **Ctrl+C**.

> Пока запущено на вашем компьютере — бот работает. Закрыли программу/выключили
> ПК — бот «спит». Для постоянной защиты нужен хостинг (следующий шаг).

Проверить, что фильтр работает, можно и без токена:
```bash
python test_spam_filter.py
```

<a name="шаг-5"></a>
## Шаг 5. Бесплатный хостинг (чтобы работал круглосуточно)

Чтобы бот работал 24/7, его нужно поставить на сервер, который не выключается.
Ниже — самый простой бесплатный путь через **Railway**. (Аналогично можно
Render или PythonAnywhere — принцип тот же.)

### Вариант А. Railway (проще всего)

1. Залейте эту папку в репозиторий на **GitHub** (можно приватный).
2. Зайдите на **railway.app**, войдите через GitHub.
3. **New Project → Deploy from GitHub repo** → выберите свой репозиторий.
4. Railway увидит `requirements.txt` и `Procfile` и сам всё соберёт.
5. Откройте вкладку **Variables** и добавьте переменную:
   - `BOT_TOKEN` = ваш токен от BotFather
   - (позже добавите `ADMIN_CHAT_ID` — см. [Шаг 6](#шаг-6))
6. Убедитесь, что сервис запускается как **worker** (Procfile это задаёт).
7. Готово — бот работает постоянно. В логах Railway увидите
   `Бот запущен. Жду сообщения…`.

> 💡 У бесплатного тарифа Railway есть месячный лимит часов. Боту нужно совсем
> немного ресурсов, обычно хватает. Если лимит закончится — есть Render (тоже
> бесплатный worker) или дешёвый VPS за ~150–300 ₽/мес.

### Вариант Б. Свой сервер / VPS (если он у вас есть)

```bash
git clone <ваш-репозиторий>
cd telegram-spam-bot
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
export BOT_TOKEN=ваш_токен
export ADMIN_CHAT_ID=ваш_id
python bot.py
```
Чтобы бот не выключался после закрытия консоли, запускайте его как сервис
(systemd) или через `screen`/`tmux`. Могу расписать отдельно, если нужно.

<a name="шаг-6"></a>
## Шаг 6. Уведомления вам в личку (ADMIN_CHAT_ID)

Чтобы бот присылал вам подозрительные сообщения с кнопками «Забанить / Не спам»:

1. Найдите своего бота в Telegram по username и нажмите **Start**
   (напишите ему `/start`).
2. Бот ответит и покажет ваш **chat_id** (число).
3. Впишите это число в настройку **ADMIN_CHAT_ID**:
   - на Railway/Render — в разделе **Variables**;
   - локально — в файле `.env`.
4. Перезапустите бота.

Теперь всё подозрительное будет прилетать вам в личку. Нажали «✅ Не спам» —
человек попадает в белый список, и бот больше его не трогает.

> Если оставить `ADMIN_CHAT_ID` пустым — бот всё равно работает: явный спам
> удаляет и банит, просто не шлёт вам уведомлений.

<a name="настройка-фильтра"></a>
## ⚙️ Как настраивать строгость фильтра

Всё, что можно крутить, лежит в двух файлах:

### В `spam_filter.py` (вверху файла):

- `FLAG_THRESHOLD = 3` — с какого числа баллов сообщение считается
  подозрительным. **Меньше = строже.**
- `BAN_THRESHOLD = 6` — с какого числа баллов — сразу бан. **Меньше = строже.**
- Списки слов `ADULT_WORDS`, `DM_WORDS`, `MONEY_WORDS`, `BAIT_WORDS` —
  можете дополнять своими словами (пишите в нижнем регистре). Например, если
  какой-то спам проходит — добавьте характерную фразу из него.

После правок запустите `python test_spam_filter.py`, чтобы убедиться, что
обычные сообщения по-прежнему проходят.

### В настройках (`.env` или Variables на хостинге):

- `DELETE_ON_SUSPICION=true` — подозрительное удаляется сразу (рекомендуется).
  Поставьте `false`, если хотите, чтобы бот только уведомлял, а удаляли вы сами.
- `TRUST_AFTER_MESSAGES=3` — сколько нормальных сообщений должен написать
  человек, чтобы бот перестал его проверять. Больше = осторожнее к новичкам.

**Хотите поймать конкретный спам, который просочился?** Скопируйте пару
характерных фраз из него в подходящий список слов в `spam_filter.py` —
и в следующий раз бот его поймает.

<a name="faq"></a>
## ❓ Частые вопросы и проблемы

**Бот добавлен, но ничего не удаляет.**
→ 99% случаев — у бота нет прав «Удаление сообщений» и «Блокировка
пользователей». Проверьте Шаг 3. Ещё проверьте, что бота добавили именно в
**группу обсуждений**, а не в канал.

**Бот забанил нормального человека.**
→ Нажмите в уведомлении «↩️ Разбанить» — он вернётся и попадёт в белый список.
Если такое повторяется — поднимите `BAN_THRESHOLD` (например до 8) и/или
уберите слишком «широкое» слово из списков.

**Спам всё равно проходит.**
→ Понизьте пороги (`FLAG_THRESHOLD`/`BAN_THRESHOLD`) или добавьте фразы из
этого спама в списки слов. Спамеры меняют тексты — списки полезно иногда
пополнять.

**Бот банит тех, кто оставляет ссылки.**
→ Это защита: ссылки от новичков — частый спам. Постоянные участники (после
`TRUST_AFTER_MESSAGES` нормальных сообщений) и админы проверку не проходят,
их ссылки не трогаются.

**Не могу удалять аватарки/картинки 18+ по самой картинке.**
→ Бот не «смотрит» содержимое картинок (это отдельная сложная технология).
Но он ловит таких спамеров по другим признакам: текст/подпись 18+, ссылки,
поведение новичка. Обычно этого достаточно, чтобы отсеять их до того, как
они начнут вредить.

**База данных обнулилась после передеплоя на хостинге.**
→ У бесплатных хостингов диск часто «эфемерный». Это не критично: бот заново
наберёт доверие к участникам. Для надёжности можно подключить постоянный диск
(volume) и указать в нём путь через переменную `DB_PATH`.
